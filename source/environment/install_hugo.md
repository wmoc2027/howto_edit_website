# Hugoのインストール

hugoとはgoで作られたwebサイトを作成するフレームワークです。gitリポジトリで構成されているファイルは、hugoにより処理される設定ファイルやmarkdownという書式で記述されたwebサイトの原稿です。この原稿から、[https://wmoc2027.jp/](https://wmoc2027.jp/) のようなHTMLに変換出力するフレームワークがhugoの役割です。

githubの [wmoc2027/public_website](https://github.com/wmoc2027/public_website) リポジトリに反映すると、githubが提供するサーバスクリプト Actions により、リポジトリツリー内の `.github/workflows/hugo.yml` 設定ファイルに基づいて自動的にHTMLファイルへ変換します。また、しかるべき場所に作成されたhtmlファイル群を配置（これをデプロイと呼びます）することで、公式WEBサイトとして公開されます。

前述の通り、この変換ソフトウェアはgithubのActionsによってhtmlファイルを出力しますので編集者のPC上にはhugoのインストールは本来必要ありません。しかし、編集した原稿や設定内容が正しく変換されるとは限りません。誤った書式や設定により意図しない結果となった場合、その修正を行ってgitへ履歴を記憶した上でgithub上に反映する必要があります。こういった修正履歴は文章や図の変更といった意図した内容の変更ではない汚れた修正履歴といえます。

これを防ぐため、ローカルPC上にもhugoをインストールし、変換されたhtmlをプレビューして確認・修正を行い、最終的に正しい変更のみを変更履歴としてgit上に変更を記録する必要があります。このため、各々の編集者のローカルPC上にもhugoをインストールし、都度変換されたwebサイトをブラウザで閲覧できるようにします。
